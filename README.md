Симуляция сбора ресурсов двумя фракциями.

В проекте 4 сцены:
- Bootstrap - первоначальная загрузка и инициализация сервисов;
- Menu - просто кнопка начать игру;
- GamePlay - непосредственно игровой геймплей;
- Restart - сцена заглушка для перезапуска игры.

В каждой сене есть EntryPoint, чтобы понятно было как она стартует.

В режиме редактора в PlayMode можно перейти из любой сцены(Editor скрипт в Bootstrap переводит). 

В проекте используется Zenject для внедрения зависимостей.

Шина событий для общения между сервисами.

Глобальная машина состояний(определяет логику перехода между сценами), возможно тут это излишне.

Для навигации дронов используется NavMesh юнити.

Графика выполнена на примитивах.

Ui(по сути только HUD в геймплее) постоен на MVVM/MVB паттерне(скорее на моем его понимании).

Что реализовано:
1. Пресет данных (ScriptableObject):
	- GamePrefsPreset (мин/макс количество дронов и скорости, частотота появления ресурсов);
	- RewardResourcePreset (префаб ресурса с его данными - награда, время добычи);
	- 
2. Ресурсы:
	- 3 вида (отличается время добычи и награда):
	- хранятся в пуле объектов;
	- при загрузке сцены в пул добавляется по односу каждого вида;
	- управляет ими RewardResourceControl (хранит список активных ресурсов, спавнит через заданный промежуток времени,						
      принимает после добычи дроном).
	
3. Дроны:
	- разделены на две фракции(крсные/синие);
	- хранятся в пуле объектов;
	- состояния реализованы через State Machine;
	- общение модели и скина происхрдит через события(модель отправляет, скин получает);
	- управляет ими DroneControl (слушает события изменеия количества и корректирует).
	
4. Скины дронов:
	- отличаютя по цвету для уаждой фракции;
	- имеют индикацию в зависимости от состояния;
	- при сборе ресурса отображает оставшееся время(подворачивается к камере);
	- при сдаче ресурса анимируется база.

5. Визуализация пути - сделана с помощью LineRenderer(у каждой фракции свой цвет).

6. Управление:
	- "WASD" для перемещения;
	- мышь для поворота;
	
7. HUD:
	- изменеие количества/скорости дронов отдельно для каждой фракции (слайдер);
	- изменеие частоты появления ресурсов (ввод текста);
	- чекбокс показвать/нет путь;
	- скорости симуляции (слайдер).
	- текущий счет для каждой команды.
